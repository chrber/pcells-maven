The CellLogin

Managing Sessions

Creating a session
Navigate from the Menu Bar

Session -> New ...

CellLogin will prompt for the name of the newly to create session. The session name might be anything which remindes you of the dCache instance this session should be connected to. Session names should be unique. After entring a valid session name, type 'ok'.

CellLogin will switch to the Cell Login panel. Chose the Setup button to custumize the Session parameters.

CellLogin offers a set of setup cards.

The Addresses card covers the server parameters. Host, port, user name and protocol. Server name is the name of the headnode of the selected dCache instance. Port Number is the ssh resp. admin port number. User name is obvious and the protocol should be ssh1. Activate the Apply button to make all changes permanent.

The Module card determines the dCache communication modules which should be loaded for this session. The CellLogin jar file is coming with a proper configured Modules card.

Adding a dCache communication module

Activate the Next button until the end of the list is reached ,at which point the Next button appears disabled. Type Insert. Chose a name for the new entry. (Arbitrary). Add the full class name of the module which should be made known under the name chosen above. Activate the previous button followed by the Apply button.

Choose the Quit button to get back to the Login panel. CellLogin stores all configured parameters together with the session name. A restarted CellLogin application will recall all configured sessions.
Connecting a session
CellLogin Panel provides two 'ssh1' methods to log into a dCache server. If no password is specified, CellLogin assumes to find a valid private rsa1 key at ${home}/.ssh/identity, otherwise, if a password is specified, CellLogin tries to login with the given 'username' 'password pair. In case the login succeeded, CellLogin offers all modules defined within the 'setup' step as stacked cards.
Chosing a predefined Session
Navigate from the Menu Bar

Session -> Open -> <session name>

CellLogin will present the chosen session.

Switching sessions
Navigate from the Menu Bar

Windows -> <session name>

CellLogin moves the selected session to front.

The Commander (org.pcells.services.gui.XCommander)

XCommander is a generic tool to communicate with any CellRealm. It's not aware of Cell applications (e.g. dCache). If XCommander is configured in the session setup, it will show up as a card after logged in.

When activated, the XCommander shows up with its logo.

Left Mouse click on the logo will switch to the XCommander main menu.

Right Mouse click will launch the Cell Domain scanner. The scanner first checks it a Topology cell can be accessed. If yes, the Topology cell is ask to provide the Domain Topology. It no TopoCell is found, the XCommander initiates a topology scan. In either case the XCommander will display the result.
