Introduction

The pCells GUI is a graphical framework, allowing to connected to one or more Cell Systems simultaniously. The framework provides the security mechnisms for login. Once configured, it stores the necessary parameters to connect to the actual cell system(s) between logins. Finally it easily allows to switch between active sessions (different cell systems). After logged in, the framework loads modules which are specific to the particular cell system, in this case the dCache. A set of plug-ins is coming with the distribution, other may be provided and loaded by the user. (See Adding your own plug-in, somewhere at the end of this help facility).

Managing Sessions

Defining a session
Navigate from the Menu Bar

Session -> New ...

CellLogin will prompt for the name of the newly to create session. The session name might be anything which remindes you of the dCache instance this session should be connected to. Session names should be unique. After entring a valid session name, type 'ok'.

CellLogin will switch to the Cell Login panel. Choose the Setup button to custumize the Session parameters.

CellLogin offers a set of setup cards.

The Addresses card covers the server parameters. Host, port, user name and protocol. Server name is the name of the headnode of the selected dCache instance. Port Number is the ssh resp. admin port number. Its the same is the ssh port number for the line command login. User name is obvious and the protocol must be ssh1. Activate the Apply button to make all changes permanent.

The Module card determines the dCache communication modules which should be loaded for this session. The CellLogin jar file is coming with a proper configured Modules card. Please refer to the Adding your own plug-in chapter to learn more about loading your own modules.

Adding a dCache communication module

Activate the Next button until the end of the list is reached ,at which point the Next button appears disabled. Type Insert. Chose a name for the new entry. (Arbitrary). Add the full class name of the module which should be made known under the name chosen above. Activate the previous button followed by the Apply button.

Choose the Quit button to get back to the Login panel. CellLogin stores all configured parameters together with the session name. A restarted CellLogin application will recall all configured sessions.

Connecting a session
CellLogin Panel provides two 'ssh1' methods to log into a dCache server. If no password is specified, CellLogin assumes to find a valid private rsa1 key at ${home}/.ssh/identity, otherwise, if a password is specified, CellLogin tries to login with the given 'username' 'password pair. In case the login succeeded, CellLogin offers all modules defined within the 'setup' step as stacked cards.
Chosing a predefined Session
Navigate from the Menu Bar

Session -> Open -> <session name>

CellLogin will present the chosen session.

Switching sessions
Navigate from the Menu Bar

Windows -> <session name>

CellLogin moves the selected session to front.
